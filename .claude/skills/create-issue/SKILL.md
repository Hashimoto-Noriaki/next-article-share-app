# Create Issue

ユーザーの説明をもとに GitHub Issue を作成する。

## Steps

1. ユーザーから以下を聞く（既に提供されている場合はスキップ）：
   - **何をしたいか・何が問題か**（Issue の内容）
   - **種別**：バグ報告 / 機能追加 / その他

2. 内容をもとに以下の形式で Issue を生成する。

## Issue フォーマット

### バグ報告

```markdown
## 概要

<!-- バグの内容を1〜2行で -->

## 再現手順

1. 
2. 
3. 

## 期待する動作

## 実際の動作

## 環境

- OS: 
- ブラウザ / Node.js バージョン: 
```

### 機能追加 / その他

```markdown
## 概要

<!-- やりたいことを1〜2行で -->

## 背景・目的

## 実装案（任意）

```

## 実行

```sh
gh issue create --title "<タイトル>" --body "$(cat <<'EOF'
<生成した本文>
EOF
)" --label "<label>"
```

### ラベルの選び方

| 内容       | label           |
| ---------- | --------------- |
| バグ報告   | `bug`           |
| 機能追加   | `enhancement`   |
| その他     | （省略）        |

作成後、Issue の URL を表示する。
